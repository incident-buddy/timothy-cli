Timothy

LLMが生成したHTMLをターミナルから一発でアップロード。
有効期限付きURLで、URLを知っている人だけに共有できる。

$ llm generate report | tim upload --stdin → https://your-api/s/01JWXYZ...

主な機能

📤

ファイル / stdin アップロード

HTMLファイルを直接、またはLLMの出力をパイプで渡してアップロード

⏱️

有効期限付きURL

デフォルト7日間。--ttl で日数を指定可能

📋

一覧・削除

tim list / tim delete でファイル管理

🔐

セルフホスト

ストレージとアクセス制御を自分のインフラで完全管理

🌐

IP制限（オプション）

ALLOWED_IPS で共有URLへのアクセスをCIDR指定で制限

CLIデモ

bash — timothy
# 1. セットアップ
$ tim setup
API key: ··················
API endpoint: https://your-api.example.com
# 2. LLM出力をパイプして共有
$ claude -p "分析レポート作成" \
| tim upload --stdin --title "分析" --ttl 30
✓ https://your-api/s/01JWXYZ...
# 3. 一覧表示
$ tim list
ID TITLE CREATED EXPIRES
01JWXYZ... 分析 2026-05-20 2026-06-19
# 4. 削除
$ tim delete 01JWXYZ...
Delete 01JWXYZ...? [y/N] y
✓ Deleted 01JWXYZ...

デプロイオプション

コードは同一

☁️ Google Cloud Run 推奨

コンテナをCloud Runにデプロイ。コールドスタートはあるが低コスト。

Compute Cloud Run（min 0 / max 2インスタンス）
Storage Firebase Cloud Storage（非公開バケット）
DB Firestore（メタデータ管理）
Secret Secret Manager（認証情報）

🟠 AWS Lambda

Firebase バックエンドはそのまま、コンピュート層のみAWSに切替。

Compute Lambda（コンテナイメージ + Function URL）
Storage Firebase Cloud Storage（共用可）
DB Firebase Firestore（Cloud Run版と共用可）
Secret AWS Secrets Manager（認証情報）

セキュリティモデル

🔑 アップロード/管理 Bearer APIキー認証（管理者が発行）
🔗 共有URL閲覧 URLを知っている人のみ（ログイン不要）
🚫 直接ストレージアクセス Cloud Storageへの直接アクセスは禁止
🌍 IP制限 ALLOWED_IPS でCIDR制限（オプション）
⏰ 有効期限 TTL経過後は失効・アクセス不可
📜 Firestoreルール クライアントからの直接書き込みを禁止

インストール

グローバルインストール

$ npm install -g timothy-cli

インストール不要で実行

$ npx timothy-cli <command>

✅  Google Cloud プロジェクト（課金有効）

✅  Docker（イメージビルド用）
✅  @timothy/api のデプロイ済みインスタンス
